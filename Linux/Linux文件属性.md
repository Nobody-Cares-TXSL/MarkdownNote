# Linux 文件属性

## 核心命令

| 命令 | 全称 | 作用 |
|------|------|------|
| `chown` | change owner | 修改文件所属用户与组 |
| `chmod` | change mode | 修改文件访问权限 |

## 查看文件属性

使用 `ls -l` 命令查看文件的详细属性：

```
-rw-rw-r-- 1 duan duan 1090  3月 20 09:46  README.md
```

各部分含义：

| 部分 | 值 | 说明 |
|------|----|------|
| 文件类型与权限 | `-rw-rw-r--` | 第 0 位文件类型，第 1-9 位权限（详见下文） |
| 硬链接数 | `1` | 连接到该文件的硬链接数量 |
| 属主 | `duan` | 文件所有者 |
| 属组 | `duan` | 文件所属的用户组 |
| 文件大小 | `1090` | 文件大小（字节） |
| 修改时间 | `3月 20 09:46` | 文件最后修改时间 |
| 文件名 | `README.md` | 文件或目录名称 |

### 文件类型（第 0 位）

| 标识 | 类型 |
|------|------|
| `d` | 目录 |
| `-` | 普通文件 |
| `l` | 链接文档（link file） |
| `b` | 块设备文件（可随机存取，如硬盘） |
| `c` | 字符设备文件（一次性读取，如键盘、鼠标） |

### 权限位（第 1-9 位）

后续 9 个字符分为 3 组，每组 3 位，均为 `rwx` 的组合：

| 位置 | 归属 | 说明 |
|------|------|------|
| 第 1-3 位 | 属主（owner） | 文件所有者的权限 |
| 第 4-6 位 | 属组（group） | 所有者同组用户的权限 |
| 第 7-9 位 | 其他用户（others） | 其他所有人的权限 |

权限字符含义：

| 字符 | 含义 | 对文件的权限 | 对目录的权限 |
|------|------|-------------|-------------|
| `r` (read) | 可读 | 查看文件内容 | 列出目录下的文件 |
| `w` (write) | 可写 | 修改文件内容 | 在目录中创建/删除文件 |
| `x` (execute) | 可执行 | 作为程序运行 | 进入该目录（`cd`） |
| `-` | 无权限 | — | — |


> - **root 用户**：文件的权限对其不起作用，root 拥有所有权限
> - 权限位固定为 `rwx` 顺序，缺少的权限用 `-` 占位

## 更改文件属性

### chgrp：更改文件属组

```
chgrp [-R] 属组名 文件名
```

- `-R`：递归更改，目录下所有文件的属组都会变更

### chown：更改文件所有者（可同时更改属组）

```
chown [-R] 所有者 文件名
chown [-R] 所有者:属组名 文件名
```

示例：

```bash
chown bin install.log          # 仅改所有者为 bin
chown root:root install.log    # 同时改所有者和属组为 root
```

### chmod：更改文件权限

两种方式：**数字法** 和 **符号法**。

#### 数字法

权限对应数字：`r=4  w=2  x=1`，每种身份的权限值累加：

| 权限 | 计算 | 值 |
|------|------|----|
| `rwx` | 4+2+1 | 7 |
| `r-x` | 4+0+1 | 5 |
| `r--` | 4+0+0 | 4 |
| `---` | 0+0+0 | 0 |

```
chmod [-R] xyz 文件或目录
```

- `xyz`：三组权限的数字组合（owner / group / others）

示例：

```bash
chmod 777 .bashrc    # -rwxrwxrwx，所有人可读可写可执行
chmod 754 .bashrc    # -rwxr-xr--，属主rwx，属组r-x，其他人r--
chmod 770 project/   # -rwxrwx---，属主和属组rwx，其他人无权限
```

#### 符号法

用身份符号 + 操作符 + 权限字符来修改：

| 身份 | 含义 | 操作符 | 含义 | 权限 |
|------|------|--------|------|------|
| `u` | 属主 | `+` | 加入 | `r` |
| `g` | 属组 | `-` | 除去 | `w` |
| `o` | 其他 | `=` | 设定 | `x` |
| `a` | 全部 | | | |

示例：

```bash
chmod u=rwx,g=rx,o=r test1   # 设定权限为 -rwxr-xr--
chmod a-x test1               # 除去所有人的可执行权限
chmod u+w test1               # 给属主增加写权限
```

## 链接（Link）

每个文件在磁盘上都有一个唯一的 **inode（索引节点）**，保存文件的元数据。链接的本质就是给同一个 inode 起多个名字。

### 硬链接（Hard Link）

直接指向同一个 inode，多个文件名共享相同的数据块。修改或删除任意一个，其余链接不受影响。

```bash
ln 源文件 硬链接名
```

```bash
ls -li original.txt hardlink.txt
# 12345678 -rw-r--r-- 2 user user 6 ... hardlink.txt
# 12345678 -rw-r--r-- 2 user user 6 ... original.txt
# inode 相同，硬链接数变为 2
```

**限制**：不能跨文件系统、不能链接目录、不能链接不存在的文件。

### 软链接（Symbolic Link）

独立的文件，有自己的 inode，内容是指向目标文件的路径字符串（类似 Windows 快捷方式）。目标删除后链接失效（broken link）。

```bash
ln -s 源文件 软链接名
```

```bash
ls -li original.txt softlink.txt
# 12345678 -rw-r--r-- 1 user user  6 ... original.txt
# 87654321 lrwxrwxrwx 1 user user 12 ... softlink.txt -> original.txt
# inode 不同，文件类型为 l（link）

readlink softlink.txt    # 查看软链接指向的路径
```

### 硬链接 vs 软链接

| 项目 | 硬链接 | 软链接 |
|------|--------|--------|
| 共享 inode | 是 | 否（各自独立） |
| 跨文件系统 | 不可以 | 可以 |
| 链接目录 | 不可以 | 可以 |
| 目标删除后 | 链接依然可用 | 变成 broken link |
| `ls -l` 显示 | 普通文件 | `l` 类型，带 `->` 箭头 |
| 典型用途 | 备份防误删 | 版本切换、快捷方式 |

> 生产环境推荐用**绝对路径**创建软链接，避免移动后失效。

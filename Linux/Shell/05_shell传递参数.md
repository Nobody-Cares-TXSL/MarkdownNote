# Shell 传递参数

## 基本用法

脚本内通过 `$n` 获取参数，n 为数字。
> 例如：`$1` 获取第一个参数，`$2` 获取第二个参数，以此类推。
>
> `$0` 获取脚本本身的名称。

## 特殊参数变量

| 变量 | 说明 |
|------|------|
| `$#` | 参数个数 |
| `$*` | 所有参数（作为**一个**字符串） |
| `$@` | 所有参数（每个参数**独立**字符串） |
| `$$` | 当前进程 PID |
| `$!` | 后台最后一个进程 PID |
| `$-` | Shell 当前选项 |
| `$?` | 上条命令退出状态（0=成功） |

## `$*` vs `$@` 核心区别

在**双引号**中才有区别（假设参数为 1 2 3）：

- `"$*"` → `"1 2 3"`（一个整体）
- `"$@"` → `"1" "2" "3"`（三个独立参数）

```bash
for i in "$*"; do echo $i; done   # 输出: 1 2 3（一行）
for i in "$@"; do echo $i; done   # 输出: 1\n2\n3（三行）
```

## 参数扩展语法

Shell 支持对位置参数进行切片操作，语法为 `${@:start:length}`。

### 基本语法

| 语法 | 含义 | 示例（参数：a b c d e） |
|------|------|----------------------|
| `${@:2}` | 从第2个到末尾 | b c d e |
| `${@:2:3}` | 从第2个开始取3个 | b c d |
| `${@: -3}` | 最后3个参数（注意空格） | c d e |
| `${@:0}` | 所有参数（含脚本名） | 脚本名 a b c d e |

### 实用示例

```bash
# 跳过第一个参数，复制剩余文件
target_dir="$1"
cp "${@:2}" "$target_dir"

# 处理可变参数
./script.sh /dest file1 file2 file3
# $1 = /dest
# ${@:2} = file1 file2 file3
```

> **注意**：引号至关重要！`"${@:2}"` 能正确处理含空格的文件名，不加引号会出错。

## 注意事项

- 参数含空格时，传递时必须用引号括起：`./test.sh "a b" "c d"`
- 函数传递参数时也必须加引号：`my_func "$@"`
- 校验参数是否存在：`if [ -n "$1" ]; then ... fi`
